## 🧠 How I think about product

I come from a data background, which means I default to asking *why* before asking *what*.

Before defining any feature, I want to know: what does the session data say? Where are users dropping off? What's the metric we're actually trying to move?

At NetEase, that instinct led me to build an "intelligent forecast curve" not because it was on the roadmap — but because session path analysis revealed users were manually recreating forecasts they didn't need to. That feature hit 60% adoption in its first quarter.
